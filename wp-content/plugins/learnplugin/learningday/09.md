# delete / drop table while uninstalling plugin

need for delete / drop table

```php
<?php
// delete / drop table while uninstalling plugin
function learn_plugin_uninstall() {
    global $wpdb;
    $table_name = $wpdb->prefix . 'learn_plugin_table';
    $sql = "DROP TABLE IF EXISTS $table_name";
    $wpdb->query($sql);
}

register_uninstall_hook(__FILE__, 'learn_plugin_uninstall');
```

## Explaining the Uninstall Hook Code

#### What This Code Does

It deletes the plugin's database table when the plugin is uninstalled from WordPress, and registers that cleanup function to run automatically on uninstall.

### Line by Line

```
function learn_plugin_uninstall()
```

Defines the cleanup function that will run when the plugin is uninstalled.

```php
<?php
$table_name = $wpdb->prefix . 'learn_plugin_table';

Builds the full table name, same as before (e.g. wp_learn_plugin_table).

$sql = "DROP TABLE IF EXISTS $table_name;";

```

Prepares a SQL statement to delete the table.

IF EXISTS is a safety guard — if the table doesn't exist for some reason, MySQL won't throw an error, it just silently does nothing.

```php
<?php
$wpdb->query($sql);
```

Executes the SQL query through WordPress's database object.

## The Key Line

```php
register_uninstall_hook(__FILE__, 'learn_plugin_uninstall');
```

```
register_uninstall_hook

WordPress function that registers a callback to run on uninstall
```

```
__FILE__

PHP constant — the full path of the current file. WordPress uses this to identify which plugin is being uninstalled
```

```
'learn_plugin_uninstall'

The function to call when uninstall happens
```

### When Does This Trigger?

This runs when a user goes to: WordPress Admin → Plugins → Deactivate → Delete

⚠️ Note: Simply deactivating a plugin does NOT trigger this. It only fires on full deletion from the admin panel.

### Why This Matters (Good Practice)

Cleaning up your own database tables on uninstall is considered good plugin etiquette — you leave no orphaned data behind in the user's database.

```php
<?php
// Exit if uninstall.php is not called by WordPress
if ( ! defined( 'WP_UNINSTALL_PLUGIN' ) ) {
    exit();
}

// Access the WordPress database object
global $wpdb;

// Specify the name of the custom database table, prefixed correctly
$table_name = $wpdb->prefix . 'your_custom_table_name';

// Drop the table from the database
$wpdb->query( "DROP TABLE IF EXISTS $table_name" );

// Optional: Delete any options the plugin might have stored in the wp_options table
delete_option( 'your_plugin_option_name' );
// For multisite options, use delete_site_option()
// delete_site_option( 'your_plugin_option_name' );
?>
```
