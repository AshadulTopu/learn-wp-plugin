# create database table while installing plugin

### create database table while installing plugin

```php
<?php
function learn_plugin_install()
{
    global $wpdb;
    $table_name = $wpdb->prefix . 'learn_plugin_table';
    $sql = "CREATE TABLE $table_name (
        id int(11) NOT NULL AUTO_INCREMENT,
        name varchar(255) NOT NULL,
        PRIMARY KEY  (id)
    )";
    require_once( ABSPATH . 'wp-admin/includes/upgrade.php' );
    dbDelta( $sql );
}

register_activation_hook( __FILE__, 'learn_plugin_install' );
```

### table generating code with check if table exists

function learn_plugin_create_table(){
global $wpdb;
require_once( ABSPATH . 'wp-admin/includes/upgrade.php' );

    if ( $wpdb->get_var( "SHOW TABLES LIKE '{$wpdb->prefix}learn_plugin_table'" ) != $wpdb->prefix . 'learn_plugin_table' ) {
        $sql = "CREATE TABLE {$wpdb->prefix}learn_plugin_table (
            id int(11) NOT NULL AUTO_INCREMENT,
            name varchar(255) NOT NULL,
            PRIMARY KEY  (id)
        )";
        dbDelta( $sql );
    }

}

## Explaining the Code

What This Code Does
It checks if a database table exists, and if not, creates it.

### Line by Line

global $wpdb;
Accesses WordPress's global database object, which handles all DB interactions.
The SHOW TABLES LIKE ... check Queries MySQL to see if the table already exists. If it doesn't exist, it runs the creation block. This prevents accidentally recreating/overwriting the table on every page load.

### How the Prefix Works

```
$wpdb->prefix is a value defined in your site's wp-config.php file:

$table_prefix = 'wp_'; // This is the default
```

So when you write:

```
$wpdb->prefix . 'learn_plugin_table'

```

It becomes:

```
wp_learn_plugin_table
```

The full table name in your database would be wp_learn_plugin_table.

Why the Prefix Exists

1. Multiple WordPress installs on one database — if you run two WP sites sharing the same DB, you can give them different prefixes (site1*, site2*) so their tables don't collide.
2. Security — changing from the default wp\_ makes it slightly harder for attackers to guess your table names in SQL injection attempts.
3. Convention — all WordPress core tables follow this pattern too: wp_posts, wp_users, wp_options, etc.

The Prefix Could Be Anything
If someone configured their wp-config.php like this:

```
$table_prefix = 'mysite_';
```

Then your table would be named mysite_learn_plugin_table instead. The code dynamically reads whatever prefix is set, so it always works correctly regardless of the environment.
