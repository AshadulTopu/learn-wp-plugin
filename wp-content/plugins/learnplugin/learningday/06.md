# Different between PLUGIN_DIR and PLUGIN_FILE and PLUGIN_URL

1. PLUGIN_FILE
   Definition:
   Stores the main plugin file path.

Example:
define( 'PLUGIN_FILE', **FILE** );

If your plugin main file is:
wp-content/plugins/my-plugin/my-plugin.php

Then:
PLUGIN_FILE
returns something like:
/var/www/html/wp-content/plugins/my-plugin/my-plugin.php

✅ Used for:
Plugin activation hooks
Getting plugin directory
Reference to main plugin file

Example:
register_activation_hook( PLUGIN_FILE, 'my_plugin_activate' );

2. PLUGIN_DIR
   Definition:
   Stores the absolute directory path of the plugin.

Example:
define( 'PLUGIN_DIR', plugin_dir_path( PLUGIN_FILE ) );

Output example:
/var/www/html/wp-content/plugins/my-plugin/

✅ Used for:
Including PHP files
Loading templates
Loading classes

Example:
require_once PLUGIN_DIR . 'includes/class-loader.php';

3. PLUGIN_URL
   Definition:
   Stores the URL of the plugin directory.

Example:
define( 'PLUGIN_URL', plugin_dir_url( PLUGIN_FILE ) );

Output example:
https://example.com/wp-content/plugins/my-plugin/

✅ Used for:
CSS files
JS files
Images

Example:

wp_enqueue_style(
'plugin-style',
PLUGIN_URL . 'assets/css/style.css'
);
